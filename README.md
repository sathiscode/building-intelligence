# Building Intelligence

**Building Intelligence** is my personal learning project for understanding modern deep learning from first principles.

I am building it in public so others can learn alongside me, question the explanations, suggest improvements, and contribute better ways to understand deep learning.

This project follows the evolution of modern deep learning — from simple neural networks to today’s foundation models — and tries to explain why each major architectural idea emerged.

The goal is simple:

> Understand deep learning by experimenting with models, breaking them, observing what fails, and discovering why each concept exists.

## Why This Exists

Many deep learning resources start with equations, terminology, or framework APIs.

Those are important, but I wanted to understand something more basic:

* Why do neural networks need activation functions?
* Why does backpropagation matter?
* Why do CNNs use convolution?
* Why did attention become so important?
* Why did Transformers change deep learning?
* Why do foundation models need retrieval, memory, tools, and alignment?

Instead of keeping my notes private, I am turning this learning journey into an open interactive resource.

## What This Project Is

Building Intelligence is an interactive learning project focused on modern deep learning.

It combines:

* short explanations,
* visual intuition,
* interactive sandboxes,
* guided experiments,
* failure-based learning,
* and practical deep learning concepts.

The focus is on **ideas, not frameworks**.

PyTorch, TensorFlow, JAX, and future frameworks may change. The underlying ideas — optimization, representation, attention, memory, retrieval, and learning — remain valuable.

## Learning Approach

The project is built around a simple pattern:

1. Start with a question.
2. Try the simplest possible idea.
3. Change or remove something.
4. Observe what breaks.
5. Understand why it breaks.
6. Introduce the concept that solves the problem.
7. Connect it to modern deep learning systems.

The main question behind every topic is:

> What problem does this idea solve?

## Current Status
